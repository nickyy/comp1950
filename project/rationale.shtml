Main Principles

The client’s site is a site built around content, almost exclusively text, to which users will be required to:

  • Absorb technical information
  • Have on-going ease of recalling information, even as content increases

To support this, our two main principles driving the design is ease of navigation and contextual legibility.

We decided to go with a fluid layout with right column fixed-width and left column dynamically resizable. This would allow for responsive resizing of the pages and one column to be used for contextual links.

LayoutTop

Navigation

The client’s site built around content, almost exclusively text, through which users will be required to access pertinent parts on an on-going basis. We have designed multiple ways users can efficiently reach the content they need.

Across the top of each page lies the main navigation bar. This leads the users to the main sections of the site and is consistent across all pages. In contrast, the right hand side navigation column changes to context specific links based on the page currently on. Most frequently this is the list of lecture notes by week and topic.

Additionally, each page has a search bar, functionality provided by Google Search, to allow users fast and efficient navigation to specific desired content. The placement of the search bar, in the upper right corner, allows it to be easily located, with the placement consistent with current industry practices.

We offer a sitemap to allow users an overview of the site structure. This gives a holistic view of the entire site. The link to this page can be found in the footer of every page.

Finally, in the event a user ends up at a page that doesn’t exist, we have a styled 404 page with several options to resolve the issue.

Mobile Layout

In the client’s answer to the questionnaire, they expressed a desire for a handheld friendly format for their site. To accomplish this, we added a media-query to differentiate user-agents below 420px and shift to a single column layout. This optimises the viewable space available to users who visit from smaller devices.

Various other elements are adjusted to optimise the user experience:

  • White background behind content for legibility
  • Sidebar nav background colour is changed to differentiate it from the main nav bar
  • Header logos are removed
  • Header font sizes are slightly reduced
  • Search button is changed to a search icon
  • Search field is enlarged
  • Navigational links are centred and the active link areas are expanded (ie. click near the links rather directly on the link)
  • Form fields are adjusted, with form buttons enlarged (assuming a touch device)

DesignTop

All pages were designed with semantics and search engine optimisation in mind. Relevant HTML5 tags, including <header>, <hgroup>, <nav>, <section>, <aside>, and others, were used as much as possible. In this regard we attempt to achieve a website that well qualifies itself as part of the Semantic Web and providing Universal Access.

The style for the main content of each page was designed for ease of readability and visual separation of the various components. For example, appropriate spacing was implemented to allow the user to easily identify the various hierarchical levels within each sub-section (bulleted lists, and bulleted sub-lists), and also between different content sections of the same page.

To ensure cross-browser compatibility, we started by zeroing out margin and padding on most elements, giving us a fresh starting point. We then built up the design of the site to W3C CSS3 standards while maintaining graceful backwards compatibility. A prominent example of this would be our use of rounded corners. Modern browsers will display as intended, while more historic browsers will display box corners. Mild aesthetic change that does not impact the ability of the site to perform.

A few of the pages had some unique elements we’d like to further elaborate on.

The Protosite

The Protosite, which was designed for our client Alistair Calder, proposes changes to the existing COMP 1950 (Intermediate Web Design and Development) website. This site is designed for students to be able to see at a glance all of the information about the course such as introduction, course outline, and each week’s lecture theme.

Contact Us

The Contact Us page was designed with a form, rather than individual email addresses. The use of a form allows page visitors to contact the team without the team having to expose their email addresses.

The Tumblr Blog

The Tumblr Blog micro-site was designed to duplicate exactly the look and feel of the class site. All of the main navigation links lead back to the client’s pages, building a seamless experience for the users. Sample text and images were pre-loaded in order to showcase how Client’s content would look.

Wireframes

The Wireframes page provides various adaptations of the wireframes, namely in DOC, HTML, and PNG formats. This allows for downloading or viewing of the wireframes in whatever media the user desires. The PNG images are the default view to allow quick and easy access to view the wireframes.

Sitemap

The Sitemap page stands out from the rest of the site, with no layout applied to it. This gives the content (the site map) the most amount of real estate possible given the user’s browser-agent. Every page is logically organized and linked.
